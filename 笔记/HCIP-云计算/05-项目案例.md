# 项目案例
## 项目需求
### （一）任务背景

2025年双十一期间，某电商公司由于IT业务未部署在云端，业务灵活性及抗高并发能力相对较弱，活动期间导致多次业务中断故障等原因。为了保证新一年618活动能够正常运行，打算尝试将业务部署在华为云系统（可选公有云、虚拟化平台）上。

现在你作为华为云技术工程师，用户提出了以下需求，请在满足以下需求的情况下，进行云上服务的选型及云上业务架构的设计方案输出，并最终进行业务测试的实施。
### （二）任务要求

（1）用户业务代码已提供，见下文下载链接，客户推荐使用国产系统为业务运行的操作系统；

opencart代码下载链接：

https://xunfangbei.obs.cn-north-4.myhuaweicloud.com/opencart.zip

该业务的运行环境包含

php环境：7.1.8+

mysql环境：5.7

（2）linux系统服务器要求为密钥对登录；

（3）采用存算分离架构，由于数据库访问压力过大，需要实现数据库读写分离相关特性；

（4）需要给业务服务器配置数据库分布式缓存特性来进行数据库访问加速；

（5）要求数据节点和业务节点进行网络分布部署，分别部署在不同的虚拟网络（公有云则不同VPC、虚拟化则不同DVS）中；

（6）使用相关服务来监控系统状态并收集系统日志，并转存至对象存储中；

（7）支持周期性对数据库进行的自动备份功能（用户业务每日9：00-次日5：00业务较为繁忙）；

（8）用户希望将代码文件存放于某可用于共享的服务中再通过共享的方式共享给前端业务服务器，以便更改代码配置时可实时更新给所有前端承载业务的服务器，并需要配置业务服务器开机自动挂载该共享存储空间；

（9）可针对不同业务量动态调整服务器个数，并且希望日常保持在3台服务器，同时也可以针对网络带宽负载压力自动调整网络带宽；

（10）可自动将流量分发到多台服务器并且支持自动故障切换以及会话保持；

#### 配置分布式缓存Redis

编译安装`php-redis`模块

```
cd /tmp

wget https://pecl.php.net/get/redis-5.3.7.tgz
tar -zxf redis-5.3.7.tgz

cd redis-5.3.7

phpize

./configure --with-php-config=/usr/bin/php-config

make
make install

echo "extension=redis.so" > /etc/php.d/redis.ini

systemctl restart php-fpm
```

修改配置文件

在`config.php`和`admin/config.php`末尾添加

```php
// REDIS_CONFIG
define('CACHE_HOSTNAME', '192.168.20.20');
define('CACHE_PORT', '6379');
define('CACHE_PREFIX', 'oc_');
define('CACHE_PASSWORD', 'XXXX');
```

在`system/config/default.php`中修改

将`$_['cache_engine'] = 'file';`修改为`$_['cache_engine'] = 'redis';`

添加密钥认证模块

修改`system/library/cache/redis.php`

修改如下部分

```php
public function __construct($expire) {
	$this->expire = $expire;
	
	$this->cache = new \Redis();
	$this->cache->pconnect(CACHE_HOSTNAME, CACHE_PORT);
	
	// 添加Redis密钥认证
	if (defined('CACHE_PASSWORD') && CACHE_PASSWORD) {
		$this->cache->auth(CACHE_PASSWORD);
	}
}
```

重启服务

```
cd /mnt/home

rm -rf vqmod/vqcache/*

systemctl restart php-fpm
```
